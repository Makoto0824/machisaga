Next.js + TypeScript + Tailwind CSS で、「まちサーガ茂原市チャンス」のWeb版プロトタイプを作成してください。

目的は、まずWebブラウザ上でチャンス機能を完成させることです。
その後、LINEミニアプリ / LIFF としてLINE公式アカウントのリッチメニューから開けるようにする予定です。

今回は、最初からLINE連携を完全実装する必要はありません。
ただし、後からLINEユーザーIDに差し替えやすい構造にしてください。

# 重要な方針

- まずは小さく完成させる
- 複雑な管理画面は作らない
- LINE公式クーポン連携はしない
- クーポンは独自クーポンとしてアプリ内で管理する
- 位置情報は使わない
- 店舗QRも初期版では使わない
- チャンスはLINEリッチメニューに常設される単体コンテンツという想定
- ゲームとは別軸のコンテンツとして扱う

# 技術構成

- Next.js
- TypeScript
- Tailwind CSS
- Supabase
- Vercelにデプロイ予定

# 初期版の仕様

## チャンス仕様

- ユーザーは1日3回までチャンスを回せる
- 初期Web版ではLINEユーザーIDの代わりに仮ユーザーIDを使う
- 仮ユーザーIDは `demo-user-001`
- 後からLINE LIFFのuserIdに差し替えられるように、必ず `getCurrentUserId()` のような関数を用意する
- チャンス結果はSupabaseに保存する
- 当たりの場合は、獲得クーポンとして保存する
- はずれもチャンス履歴として保存する
- 1日の残り回数を画面に表示する

## クーポン仕様

- チャンスで当たったクーポンは、アプリ内の「獲得クーポン」画面に保存される
- LINE公式クーポンではなく、まちサーガ独自クーポンとして扱う
- クーポンには有効期限を設定する
- 初期版では有効期限は発行日から7日間
- クーポンは店頭で画面提示して使う想定
- クーポン詳細画面に「使用する」ボタンを置く
- 「使用する」を押すと `used_at` を保存し、使用済み表示にする
- 使用済みのクーポンは再利用できない
- 店員用管理画面は作らない
- クーポン詳細画面には以下の注意書きを表示する

「この画面をスタッフに提示してください。内容確認後、スタッフの前で『使用する』ボタンを押してください。」

# 画面構成

以下の画面を作ってください。

## 1. ホーム / チャンス画面

- タイトル：まちサーガ茂原市チャンス
- 説明文：
  「1日3回までチャレンジできます。参加店舗で使えるクーポンや特典が当たります。」
- 残り回数表示
  例：「本日の残り回数：3回」
- 大きなチャンス演出エリア
- 「チャンスを回す」ボタン
- チャンス実行中の簡単な演出
- 結果表示
  - 当たりの場合：クーポン名、店舗名、説明
  - はずれの場合：「はずれ。また明日チャレンジ」
- 結果表示後に以下のボタン
  - 獲得クーポンを見る
  - 参加店舗を見る
  - もう一度回す

## 2. 獲得クーポン一覧画面

- タイトル：獲得クーポン
- ユーザーが獲得したクーポン一覧を表示
- 各カードに表示する内容
  - クーポン名
  - 対象店舗
  - 有効期限
  - 使用済み / 未使用
  - 詳細を見るボタン
- 期限切れのものは「期限切れ」と表示
- 使用済みのものは「使用済み」と表示

## 3. クーポン詳細画面

- クーポン名
- 対象店舗
- 特典内容
- 利用条件
- 有効期限
- 使用状況
- 注意書き
- 「使用する」ボタン
- 使用済みの場合はボタンを無効化する

## 4. 参加店舗画面

- タイトル：参加店舗
- ダミー店舗を一覧表示
- 店舗カードに以下を表示
  - 店舗名
  - 業種
  - 紹介文
  - 営業時間
  - 関連クーポン
- 初期版では詳細モーダル程度でよい

# Supabase設計

Supabaseを使う前提で実装してください。
ただし、最初にローカルや仮データでも動くようにして、Supabase接続情報がない場合は画面だけ確認できるようにしてください。

必要なテーブルは以下です。

## coupons

- id: uuid primary key
- store_id: text
- store_name: text
- title: text
- description: text
- usage_condition: text
- probability: integer
- expires_days: integer
- is_active: boolean
- created_at: timestamp

## chance_logs

- id: uuid primary key
- user_id: text
- coupon_id: uuid nullable
- result_type: text
  - win
  - lose
- played_at: timestamp

## user_coupons

- id: uuid primary key
- user_id: text
- coupon_id: uuid
- store_name: text
- title: text
- description: text
- usage_condition: text
- issued_at: timestamp
- expires_at: timestamp
- used_at: timestamp nullable

# 実装ロジック

## 現在ユーザー取得

以下のような関数を作ってください。

`getCurrentUserId()`

初期版では必ず以下を返してください。

`demo-user-001`

後でLINE LIFF対応するときに、この関数の中身だけ差し替えられるようにしてください。

## 1日3回制限

- 今日の0:00〜23:59の `chance_logs` を数える
- 3回未満なら回せる
- 3回以上なら「本日のチャンス回数を使い切りました」と表示
- 残り回数を画面に表示

## チャンス抽選

- activeなクーポンと「はずれ」を抽選対象にする
- probabilityに応じて重み付き抽選する
- 当たりなら
  - chance_logsに保存
  - user_couponsに保存
- はずれなら
  - chance_logsに保存
  - user_couponsには保存しない

## クーポン使用

- user_couponsのused_atを現在時刻で更新
- 使用済み表示にする
- 期限切れの場合は使用できない
